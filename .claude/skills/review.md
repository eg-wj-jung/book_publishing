# 리뷰 + 수정 사이클

이 스킬은 내부 리뷰(1~6단계)와 외부 리뷰(7~9단계)를 순서대로 수행한다. 각 단계가 끝나면 다음 단계로 자동 진행한다.

## 시작 전 준비

`.claude/book-toc.md`를 읽고 목차와 페르소나를 파악한다.

## 심각도 분류

모든 리뷰 단계에서 지적 사항은 아래 3단계로 분류한다:
- 🔴 필수: 반드시 수정해야 출판 가능. 사실 오류, 논리적 모순, 목차 누락, 작성 규칙 위반
- 🟡 권장: 수정하면 품질이 올라감. 표현 개선, 흐름 보완, 예시 보강
- 🟢 참고: 고치지 않아도 됨. 스타일 선호, 대안 제안

수정 단계에서는 🔴 필수를 전부 반영하고, 🟡 권장은 판단하여 반영하고, 🟢 참고는 무시해도 된다.

## 이어하기 로직

draft/ 폴더의 파일 존재 여부를 확인하여 현재 위치를 자동 판별한다.
이미 완료된 단계는 건너뛰고, 미완료 단계부터 시작한다.

```
03_draft-v1.md만 존재 → 1단계(비평)부터
04_review-red.md 존재, 05_draft-v2.md 미존재 → 2단계(수정)부터
05_draft-v2.md 존재, 06_review-pink.md 미존재 → 3단계(독자 리뷰)부터
06_review-pink.md 존재, 07_draft-v3.md 미존재 → 4단계(수정)부터
07_draft-v3.md 존재, 08_ensemble-review-1.md 미존재 → 5단계(합평)부터
11_draft-final.md 존재, 12_review-editor.md 미존재 → 7단계(외부 리뷰)부터
14_review-proofreader.md 존재, 11_draft-final.md 갱신 필요 → 9단계(최종 수정)부터
모두 완료 → 사용자에게 "/publish로 퍼블리싱하세요"라고 안내
```

감지된 시작 지점을 사용자에게 확인한 뒤 작업을 시작한다.

# 내부 리뷰 (1~6단계)

## 1단계: 비평 리뷰

냉정한 비평가로서 초안을 평가한다. 칭찬보다 개선점을 먼저 말한다.

입력: `draft/03_draft-v1.md`

평가 기준:
- 논리적 비약이 있는가 (주장에 근거가 있는가)
- 근거 없는 주장이 있는가 (소스 기반인가, 추측인가)
- 중복된 내용이 있는가
- CLAUDE.md의 작성 규칙, book-toc.md의 작성 스타일을 위반한 곳이 있는가
- 목차에 명시된 하위 섹션이 빠진 곳이 있는가

루브릭 (10점 만점):
- 🔴 근거 없는 주장 1건 -1점
- 🟡 논리 비약 1건 -0.5점
- 감점 사유를 구체적으로 적시한다 ("5부 활용법 09에서 소스 기반 근거 없이 주장" 등)

출력: `draft/04_review-red.md`

## 2단계: 수정 (비평 반영)

`.claude/book-toc.md`의 페르소나로 글을 쓴다.

입력: `draft/03_draft-v1.md` + `draft/04_review-red.md`

수정 원칙:
- 🔴 필수 항목은 전부 반영한다
- 🟡 권장 항목은 판단하여 반영한다
- 수정한 내용과 수정하지 않은 내용(및 사유)을 변경 로그로 남긴다

출력: `draft/05_draft-v2.md`

## 3단계: 독자 관점 리뷰

기술 배경이 없는 독자의 눈으로 읽는다.

입력: `draft/05_draft-v2.md`

평가 기준:
- 이해하기 어려운 설명이 있는가
- 전문 용어가 설명 없이 사용된 곳이 있는가
- 독자가 이탈할 만한 지점이 있는가
- 실제로 따라 해볼 수 있을 만큼 구체적인가

루브릭 (10점 만점):
- 🔴 용어 미설명으로 독자가 진행 불가 1건 -1점
- 🟡 난해한 설명 1건 -0.5점, 용어 미설명 1건 -0.5점

출력: `draft/06_review-pink.md`

## 4단계: 수정 (독자 리뷰 반영)

입력: `draft/05_draft-v2.md` + `draft/06_review-pink.md`
출력: `draft/07_draft-v3.md`

## 5단계: 합평 (1라운드)

5명이 각자의 전문 영역에서 원고를 평가한다. 10점 만점.

입력: `draft/07_draft-v3.md`

평가자별 루브릭:

리서처:
- 사실 정확성, 출처 누락, 데이터 불일치
- 🔴 사실 오류 1건 -1점
- 🟡 출처 누락 1건 -0.5점

아키텍트:
- 구조 논리, 목차 누락, 분량 밸런스
- 🔴 목차 누락 1건 -1점
- 🟡 분량 편차 심함 -0.5점

리뷰어:
- 논리적 비약, 근거 없는 주장, 중복
- 🔴 근거 없는 주장 1건 -1점
- 🟡 논리 비약 1건 -0.5점

예상독자:
- 이해하기 어려운 설명, 전문 용어 미설명, 독자 이탈 지점
- 🟡 난해한 설명 1건 -0.5점, 용어 미설명 1건 -0.5점

퍼블리셔:
- 서식 규칙 위반, 시각 자료 누락, 플레이스홀더 오류
- 🔴 시각 자료 누락 1건 -1점
- 🟡 서식 위반 1건 -0.5점

평가 원칙:
- 모든 지적에 🔴/🟡/🟢 심각도를 표기한다
- 감점 사유를 구체적으로 적시한다
- "전반적으로 좋다"는 식의 모호한 평가는 하지 않는다
- 결함이 없으면 10점을 부여한다

출력: `draft/08_ensemble-review-1.md`

합평 결과를 반영하여 수정 → `draft/09_draft-v4.md`

## 6단계: 합평 (2라운드)

입력: `draft/09_draft-v4.md`
출력: `draft/10_ensemble-review-2.md`

통과 기준:
- 🔴 필수가 0건이고 5명 평균 9점 이상이면 통과
- 🔴가 남아있으면 점수와 무관하게 재수정
- 9점 미만이면 수정 후 합평 1회 추가 (최대 3라운드)
- 3라운드 후에도 미통과면 현재 버전으로 진행

합평 통과 후: `draft/11_draft-final.md` 생성

# 외부 리뷰 (7~9단계)

## 7단계: 편집자 리뷰

20년차 출판사 편집자로서 평가한다. "독자가 이 책을 끝까지 읽게 만들 수 있는가"를 가장 중요하게 본다. 칭찬보다 개선점을 먼저 말한다.

모든 지적에 심각도를 표기한다:
- 🔴 필수: 반드시 수정. 구조 결함, 서사 단절, 톤 붕괴
- 🟡 권장: 수정하면 품질 상승. 흐름 개선, 분량 조절
- 🟢 참고: 고치지 않아도 됨. 스타일 선호

평가 항목: 구성력, 서사력, 톤 일관성, 분량 밸런스, 독자 경험, 시장 경쟁력, 완성도

출력: `draft/12_review-editor.md`

## 8단계: 마케터 리뷰 + 프루프리더 리뷰

마케터 (참고용, 본문 수정에 반영하지 않음):
- "이 책을 한 줄로 소개한다면?"이라는 관점
- 평가 항목: 타겟 명확성, 제목 파워, 프롤로그 흡인력, 목차 매력도, 시의성
- 출력: `draft/13_review-marketer.md`

프루프리더:
- 20년차 교정교열 전문가. 감정 없이 사실만 지적한다.
- 모든 지적에 심각도를 표기한다:
  - 🔴 필수: 맞춤법 오류, 비문, 참조 불일치
  - 🟡 권장: 어색한 표현, 용어 불통일
  - 🟢 참고: 스타일 선호 차이
- 평가 항목: 맞춤법/오타, 비문, 용어 일관성, 문체 일관성, 부호 사용, 숫자/데이터 일관성, 참조 정확성
- 각 지적 사항: 심각도, 위치, 원문, 문제점, 수정 제안 형식으로 기록
- 출력: `draft/14_review-proofreader.md`

## 9단계: 최종 수정

편집자와 프루프리더의 피드백을 반영하여 원고를 수정한다 (마케터 의견은 참고만).

- 🔴 필수 항목은 전부 반영한다
- 🟡 권장 항목은 판단하여 반영한다
- `.claude/book-toc.md`의 페르소나로 수정한다
- 수정된 본문은 `draft/11_draft-final.md`에 덮어쓴다

완료되면 사용자에게 안내한다: "리뷰 사이클 완료. `/publish`로 Word 문서를 생성하세요."
